Ambition

Right now, three years from my first written code, several projects, and work as practicing or employee in two firms, I can call myself junior front-end developer.

What does that mean?

I can work on the development of prototypes, beta versions and main/ready-to-publish versions of project visual identity, UI and UX of some small or medium project, or to be an interactive part of the team for bigger projects.

Also, I did not limit myself only to the design, dynamic, and interactivity of the web page. I learn how to established the whole project, from design to hosting, and accordingly, the web architecture is clear to me, which allows me to see the bigger picture of web application development and implement state-of-the-art features.

On the front-end part of the web page, I try to implement advanced features as well, such as some app-like functionalities that manipulates with entered data, and sending informations to server and to create the most interactive UI possible and thus provide users with the best possible experience.

My goal is to devote myself to every project as much as possible, as it enhances my professional progress.

Through personal projects, I perfect the knowledge I acquire. I am currently learning a Node.js and ReactNative to ensure that apps that I working on can be mobile apps. On my learning path is manipulation of off-line states of the app, which make apps that I'm working on Progressive Web Apps (PWA).

I feel comfortable while using the technologies I have learned so far, so I can say without hesitation that MERN stack development is my thing and I strive to develop my skills in that direction.

I strive to keep up with the latest trends in the industry such as the implementation of AI, machine learning patterns, face recognition, off line states of app, cloud based development, as well as good practice when setting up a project and writing code.

As for the technologies and tools I use, I try to work in accordance with the latest standards, so, for example, I write React code in functional components with hooks.

All this is aimed at better organization, more readable and clear code, so that more developers can get involved in the applications I work on and start developing without much lost time getting acquainted with the code base.

My biggest goal is to become an expert at the knowledge I have acquired and to improve my skills, as well as to expand my knowledge to other areas that would improve my ability to work on different projects.
